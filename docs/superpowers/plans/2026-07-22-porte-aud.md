# Porte AUD — Auditabilité : plan d'implémentation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rendre Obscura auditable par un tiers — vecteurs ACVP ciblés sur les
primitives FIPS, fixture de conformité rejouable, et `docs/` promu seule source
normative.

**Architecture:** Trois blocs indépendants. (1) Des vecteurs officiels NIST
vendorisés dans un format texte trivial, rejoués contre `pqcrypto` **en direct**
— pas à travers la couche hybride d'Obscura, parce que `kem::decapsulate` et
`sig::verify` combinent deux primitives et ne peuvent pas consommer un vecteur
brut. (2) Une fixture de conformité (genèse + bloc scellé + valeurs attendues)
rejouable par une seule commande, qui rend falsifiable le critère « un tiers
vérifie en n'ayant lu que `docs/` ». (3) Le déplacement de l'autorité normative
de `CLAUDE.md`/`AGENTS.md` vers `docs/`.

**Tech Stack:** Rust 1.87, `pqcrypto-mlkem` 0.1.x, `pqcrypto-mldsa` 0.1.x,
`pqcrypto-traits`, `hex`. **Aucune dépendance nouvelle** — en particulier pas de
`serde_json` : les vecteurs ACVP sont convertis **une fois** en format texte
ligne-à-ligne, et le convertisseur n'est pas du code de production.

**Spec de référence :**
`docs/superpowers/specs/2026-07-22-portes-vers-le-mainnet-design.md` (révision 3).

## Global Constraints

- **Code et commentaires en français.** Sans exception, y compris les tests.
- **Aucune dépendance ajoutée au workspace.** `hex` est déjà une dépendance de
  `crypto`, donc disponible dans ses tests.
- **Le build par défaut reste la surface consensus seule.** Aucun test de ce plan
  ne doit exiger `--all-features`.
- **Ne jamais écrire « vecteurs KAT » ni « KAT FIPS complets ».** Le terme est
  **« vecteurs ACVP ciblés »**, partout, y compris dans les noms de fichiers, les
  messages de test et les commits. Raison : `keyGen`, `encap` et `sigGen` ne sont
  pas couverts, et une surpromesse dans un titre est ce qu'un auditeur relève en
  premier.
- **Tailles FIPS à ne jamais coder en dur ailleurs que dans les tests de
  conformité** : ML-KEM-768 `ek` 1184, `dk` 2400, `ct` 1088, `ss` 32 ;
  ML-DSA-65 `pk` 1952, `sk` 4032, `sig` 3309.
- **MSRV 1.87**, déjà déclaré dans `Cargo.toml`.
- **Un commit par tâche**, message en français, avec `Co-Authored-By` si généré.

```rust
//! Conformité ACVP CIBLÉE — ML-KEM-768, opération de DÉCAPSULATION.
//!
//! # Pourquoi ce test appelle `pqcrypto` en direct
//!
//! `crypto::kem::decapsulate` est HYBRIDE : il fait un Diffie-Hellman X25519, une
//! décapsulation ML-KEM, puis combine les deux par KDF sur le transcript. Il ne
//! peut donc pas consommer un vecteur ACVP, qui porte sur la primitive BRUTE.
//! Ce test valide le BACKEND (`pqcrypto-mlkem`), pas la couche hybride.
//!
//! Ce n'est pas un pis-aller : le backend est précisément ce qui est marqué
//! `unmaintained` (cf. docs/BACKEND_PQ.md), donc ce test est le filet du jour où
//! il sera remplacé. Un backend substitué qui ne rejoue pas ces vecteurs est
//! refusé avant d'atteindre le consensus.
//!
//! # Ce qui n'est PAS couvert, et pourquoi
//!
//! `keyGen`, `encap` et `sigGen` exigent d'injecter l'aléa officiel.
//! `mlkem768::keypair()` ne prend AUCUN argument : l'injection est impossible
//! avec ce backend. Voir `vecteurs/PROVENANCE.md`.

use pqcrypto_mlkem::mlkem768;
use pqcrypto_traits::kem::{Ciphertext as _, SecretKey as _, SharedSecret as _};

const FICHIER: &str = include_str!("vecteurs/mlkem768-decap.txt");

/// Tailles FIPS 203 pour ML-KEM-768. Codées en dur ICI et nulle part ailleurs :
/// un vecteur officiel qui ne les respecte pas n'est pas un vecteur ML-KEM-768.
const DK_LEN: usize = 2400;
const CT_LEN: usize = 1088;
const SS_LEN: usize = 32;

/// Un vecteur : clé de décapsulation, chiffré, secret partagé attendu.
struct Vecteur {
    dk: Vec<u8>,
    c: Vec<u8>,
    k: Vec<u8>,
}

/// Charge les vecteurs. Toute ligne malformée est une ERREUR FRANCHE : un fichier
/// de vecteurs à moitié lu donnerait un test vert sur trois vecteurs au lieu de
/// cent, ce qui est pire que pas de test du tout.
fn charger() -> Vec<Vecteur> {
    let mut v = Vec::new();
    for (n, ligne) in FICHIER.lines().enumerate() {
        let ligne = ligne.trim();
        if ligne.is_empty() || ligne.starts_with('#') {
            continue;
        }
        let champs: Vec<&str> = ligne.split(':').collect();
        assert_eq!(
            champs.len(),
            3,
            "ligne {} : attendu 3 champs séparés par ':', reçu {}",
            n + 1,
            champs.len()
        );
        let dk = hex::decode(champs[0])
            .unwrap_or_else(|e| panic!("ligne {} : dk non hexadécimal ({e})", n + 1));
        let c = hex::decode(champs[1])
            .unwrap_or_else(|e| panic!("ligne {} : c non hexadécimal ({e})", n + 1));
        let k = hex::decode(champs[2])
            .unwrap_or_else(|e| panic!("ligne {} : k non hexadécimal ({e})", n + 1));
        assert_eq!(dk.len(), DK_LEN, "ligne {} : dk fait {} o", n + 1, dk.len());
        assert_eq!(c.len(), CT_LEN, "ligne {} : c fait {} o", n + 1, c.len());
        assert_eq!(k.len(), SS_LEN, "ligne {} : k fait {} o", n + 1, k.len());
        v.push(Vecteur { dk, c, k });
    }
    v
}

/// Le fichier doit contenir des vecteurs. Sans cette assertion, supprimer le
/// fichier rendrait la suite VERTE — l'échec silencieux le plus coûteux possible
/// pour un test dont l'unique raison d'être est de convaincre un auditeur.
#[test]
fn le_fichier_de_vecteurs_est_peuple() {
    let v = charger();
    assert!(
        v.len() >= 10,
        "au moins 10 vecteurs attendus, {} trouvés — voir vecteurs/PROVENANCE.md",
        v.len()
    );
}

#[test]
fn acvp_mlkem768_decapsulation() {
    for (i, vec) in charger().iter().enumerate() {
        // `{e:?}` et non `{e}` : l'erreur de `pqcrypto-traits` garantit `Debug`,
        // pas `Display`. Un plan qui suppose l'inverse ne compile pas.
        let sk = mlkem768::SecretKey::from_bytes(&vec.dk)
            .unwrap_or_else(|e| panic!("vecteur {i} : dk refusé par le backend ({e:?})"));
        let ct = mlkem768::Ciphertext::from_bytes(&vec.c)
            .unwrap_or_else(|e| panic!("vecteur {i} : c refusé par le backend ({e:?})"));
        let ss = mlkem768::decapsulate(&ct, &sk);
        assert_eq!(
            ss.as_bytes(),
            &vec.k[..],
            "vecteur {i} : secret partagé différent du vecteur officiel"
        );
    }
}
```

- [ ] **Step 3: Écrire le convertisseur**

Créer `outils/convertir-acvp.py`. Ce fichier n'entre dans aucun build ; il
documente la dérivation pour qu'elle soit reproductible.

```python
#!/usr/bin/env python3
"""Convertit les JSON ACVP du NIST en vecteurs texte pour crates/crypto/tests.

Usage :
    python outils/convertir-acvp.py mlkem-decap  <prompt.json> <expectedResults.json>
    python outils/convertir-acvp.py mldsa-sigver <prompt.json> <expectedResults.json>

Le résultat part sur stdout ; le rediriger vers le fichier de vecteurs.

POURQUOI un format texte plutôt que le JSON tel quel : le dépôt n'a pas de
dépendance JSON et n'en veut pas dans le consensus. La conversion a lieu UNE
FOIS, hors build, et le résultat est vendorisé.
"""
import json
import sys


def charger(prompt_path, results_path):
    with open(prompt_path, encoding="utf-8") as f:
        prompt = json.load(f)
    with open(results_path, encoding="utf-8") as f:
        results = json.load(f)
    # Les résultats attendus sont indexés par tcId, pas dans l'ordre du prompt.
    attendus = {}
    for groupe in results["testGroups"]:
        for t in groupe["tests"]:
            attendus[t["tcId"]] = t
    return prompt, attendus


def mlkem_decap(prompt, attendus):
    for groupe in prompt["testGroups"]:
        # Seuls les groupes de DÉCAPSULATION nous concernent : l'encapsulation
        # tire de l'aléa que le backend ne laisse pas injecter.
        if groupe.get("function") != "decapsulation":
            continue
        dk = groupe["dk"]
        for t in groupe["tests"]:
            att = attendus.get(t["tcId"])
            if att is None or "k" not in att:
                continue
            print(f"{dk}:{t['c']}:{att['k']}")


def mldsa_sigver(prompt, attendus):
    for groupe in prompt["testGroups"]:
        # Ne garder que la variante que le backend implémente : signature PURE,
        # sans pré-hachage. Tout autre groupe est EXCLU et compté, jamais ignoré
        # en silence — cf. PROVENANCE.md.
        if groupe.get("preHash") not in (None, "none"):
            continue
        pk = groupe["pk"]
        for t in groupe["tests"]:
            att = attendus.get(t["tcId"])
            if att is None or "testPassed" not in att:
                continue
            attendu = "1" if att["testPassed"] else "0"
            print(f"{pk}:{t['message']}:{t['signature']}:{attendu}")


def main():
    if len(sys.argv) != 4:
        print(__doc__, file=sys.stderr)
        sys.exit(2)
    quoi, prompt_path, results_path = sys.argv[1], sys.argv[2], sys.argv[3]
    prompt, attendus = charger(prompt_path, results_path)
    if quoi == "mlkem-decap":
        mlkem_decap(prompt, attendus)
    elif quoi == "mldsa-sigver":
        mldsa_sigver(prompt, attendus)
    else:
        print(f"inconnu : {quoi}", file=sys.stderr)
        sys.exit(2)


if __name__ == "__main__":
    main()
```

```markdown
# Provenance des vecteurs de conformité

## Ce qui est couvert

| Primitive | Opération | Fichier |
|---|---|---|
| ML-KEM-768 (FIPS 203) | **décapsulation** | `mlkem768-decap.txt` |
| ML-DSA-65 (FIPS 204) | **vérification de signature** | `mldsa65-sigver.txt` |

Ce sont **exactement les deux opérations que le consensus exécute** : un nœud
vérifie des signatures et décapsule. Il ne rejoue jamais la génération de clés
d'autrui, ni l'encapsulation d'autrui, ni la signature d'autrui.

## Ce qui n'est PAS couvert, et pourquoi

`keyGen`, `encap` et `sigGen` exigent d'injecter l'aléa officiel du vecteur.
`pqcrypto_mlkem::mlkem768::keypair()` **ne prend aucun argument** et
`pqcrypto_mldsa::mldsa65::keypair()` non plus : le backend n'expose aucune
variante déterministe. Ces opérations sont donc **non vérifiables par vecteurs
officiels avec ce backend**.

**Ce trou est un critère de re-test du backend PQ** (voir `docs/BACKEND_PQ.md`) :
un backend exposant l'injection d'aléa le refermerait, et c'est une raison
positive d'en changer le jour venu.

⚠️ **Ne jamais décrire ce répertoire comme des « KAT FIPS complets ».** Il
contient des **vecteurs ACVP ciblés**. La différence est exactement ce qu'un
auditeur vérifiera en premier.

## Groupes exclus à la conversion

Les groupes ML-DSA à pré-hachage (`preHash` autre que `none`) sont exclus : le
backend implémente la variante pure. Le nombre de groupes exclus est reporté
ci-dessous et doit être mis à jour si les vecteurs sont régénérés.

- Groupes ML-KEM retenus (décapsulation) : _à remplir à la conversion_
- Groupes ML-DSA retenus (pure) : _à remplir à la conversion_
- Groupes ML-DSA exclus (pré-hachage) : _à remplir à la conversion_
```

Créer `crates/crypto/tests/acvp_mldsa65.rs` :

```rust
//! Conformité ACVP CIBLÉE — ML-DSA-65, opération de VÉRIFICATION.
//!
//! Même raison qu'en ML-KEM d'appeler `pqcrypto` en direct : `crypto::sig::verify`
//! est hybride (Ed25519 ET ML-DSA) et encadre le message par domaine et
//! identifiant d'algorithme (`frame`). Un vecteur ACVP porte sur le message BRUT.
//!
//! Ces vecteurs contiennent des signatures VOLONTAIREMENT INVALIDES : le test
//! vérifie l'acceptation et le REJET. Sans les cas négatifs, une fonction qui
//! retourne toujours « valide » passerait.

use pqcrypto_mldsa::mldsa65;
use pqcrypto_traits::sign::{DetachedSignature as _, PublicKey as _};

const FICHIER: &str = include_str!("vecteurs/mldsa65-sigver.txt");

/// Tailles FIPS 204 pour ML-DSA-65.
const PK_LEN: usize = 1952;
const SIG_LEN: usize = 3309;

struct Vecteur {
    pk: Vec<u8>,
    msg: Vec<u8>,
    sig: Vec<u8>,
    /// `true` si le vecteur officiel attend une signature VALIDE.
    attendu: bool,
}

/// Même discipline qu'en ML-KEM : toute ligne malformée est une erreur franche.
fn charger() -> Vec<Vecteur> {
    let mut v = Vec::new();
    for (n, ligne) in FICHIER.lines().enumerate() {
        let ligne = ligne.trim();
        if ligne.is_empty() || ligne.starts_with('#') {
            continue;
        }
        let champs: Vec<&str> = ligne.split(':').collect();
        assert_eq!(
            champs.len(),
            4,
            "ligne {} : attendu 4 champs séparés par ':', reçu {}",
            n + 1,
            champs.len()
        );
        let pk = hex::decode(champs[0])
            .unwrap_or_else(|e| panic!("ligne {} : pk non hexadécimal ({e})", n + 1));
        let msg = hex::decode(champs[1])
            .unwrap_or_else(|e| panic!("ligne {} : message non hexadécimal ({e})", n + 1));
        let sig = hex::decode(champs[2])
            .unwrap_or_else(|e| panic!("ligne {} : signature non hexadécimale ({e})", n + 1));
        let attendu = match champs[3] {
            "1" => true,
            "0" => false,
            autre => panic!("ligne {} : attendu '0' ou '1', reçu {autre:?}", n + 1),
        };
        assert_eq!(pk.len(), PK_LEN, "ligne {} : pk fait {} o", n + 1, pk.len());
        assert_eq!(
            sig.len(),
            SIG_LEN,
            "ligne {} : signature fait {} o",
            n + 1,
            sig.len()
        );
        v.push(Vecteur {
            pk,
            msg,
            sig,
            attendu,
        });
    }
    v
}

#[test]
fn le_fichier_de_vecteurs_est_peuple() {
    let v = charger();
    assert!(
        v.len() >= 10,
        "au moins 10 vecteurs attendus, {} trouvés — voir vecteurs/PROVENANCE.md",
        v.len()
    );
}

/// Le jeu doit contenir des cas NÉGATIFS. Sans eux, le test ne distingue pas une
/// implémentation correcte d'une fonction qui accepte tout.
#[test]
fn le_jeu_contient_des_cas_negatifs() {
    let v = charger();
    let negatifs = v.iter().filter(|x| !x.attendu).count();
    assert!(
        negatifs > 0,
        "aucun vecteur à signature invalide : le jeu ne teste pas le REJET"
    );
}

#[test]
fn acvp_mldsa65_verification() {
    for (i, vec) in charger().iter().enumerate() {
        // `{e:?}` : voir la note dans `acvp_mlkem768.rs`.
        let pk = mldsa65::PublicKey::from_bytes(&vec.pk)
            .unwrap_or_else(|e| panic!("vecteur {i} : pk refusée par le backend ({e:?})"));
        // Une signature invalide peut l'être par sa STRUCTURE : le décodage
        // échoue alors, et c'est un rejet légitime — pas un défaut du test.
        let obtenu = match mldsa65::DetachedSignature::from_bytes(&vec.sig) {
            Ok(sig) => mldsa65::verify_detached_signature(&sig, &vec.msg, &pk).is_ok(),
            Err(_) => false,
        };
        assert_eq!(
            obtenu, vec.attendu,
```

//! FIXTURE DE CONFORMITÉ — la commande qu'un tiers rejoue.
//!
//! Le critère de la porte AUD est « un tiers vérifie un bloc de la chaîne en
//! n'ayant lu que `docs/` ». Tel quel, il est intestable. Cette fixture le rend
//! falsifiable en une exécution : soit la commande sort les identifiants et les
//! racines attendus, soit elle ne les sort pas.
//!
//! Contenu : une genèse à UNE autorité (donc une chaîne fermée, où le scellement
//! est obligatoire) et un bloc VIDE de hauteur 1, scellé par cette autorité.
//! Aucune transaction, donc aucune preuve STARK — la fixture reste petite et
//! rapide, tout en exerçant chaînage, élection de producteur, vérification de
//! scellement et avancée de la tête.
//!
//! Le générateur (`generer_la_fixture`, `#[ignore]`) produit les fichiers ; il
//! n'est lancé qu'à la main, et son résultat est versionné.

use crypto::sig::SigKeypair;
use ledger::bloc::Bloc;
use ledger::proved_state::ProvedLedgerState;
use std::path::PathBuf;

fn racine_fixture() -> PathBuf {
    PathBuf::from(env!("CARGO_MANIFEST_DIR"))
        .join("../../docs/fixtures/conformite-v1")
}

fn lire(nom: &str) -> Vec<u8> {
    let p = racine_fixture().join(nom);
    std::fs::read(&p).unwrap_or_else(|e| panic!("fixture illisible {} : {e}", p.display()))
}

/// Lit `attendu.txt` : des lignes `clé=valeur_hex`, `#` en commentaire.
fn attendus() -> std::collections::BTreeMap<String, String> {
    let texte = String::from_utf8(lire("attendu.txt")).expect("attendu.txt non UTF-8");
    let mut m = std::collections::BTreeMap::new();
    for ligne in texte.lines() {
        let ligne = ligne.trim();
        if ligne.is_empty() || ligne.starts_with('#') {
            continue;
        }
        let (cle, val) = ligne
            .split_once('=')
            .unwrap_or_else(|| panic!("ligne sans '=' : {ligne:?}"));
        m.insert(cle.trim().to_string(), val.trim().to_string());
    }
    m
}

#[test]
fn la_fixture_se_rejoue() {
    let att = attendus();

    // 1. La genèse se décode et son identifiant est celui qui est publié.
    let genese = Bloc::from_bytes(&lire("genese.bin")).expect("genèse indécodable");
    assert_eq!(
        hex::encode(genese.id()),
        att["genese_id"],
        "l'identifiant de genèse diffère de la valeur publiée"
    );

    // 2. Elle amorce, et la racine d'état est celle qui est publiée.
    let mut etat = ProvedLedgerState::depuis_genese(&genese).expect("genèse inamorçable");
    assert_eq!(
        hex::encode(etat.tree.root()),
        att["racine_apres_genese"],
        "racine après genèse différente"
    );
    assert_eq!(hex::encode(etat.tete()), att["genese_id"], "tête ≠ genèse");

    // 3. Le bloc 1 se décode, son identifiant est celui qui est publié, et son
    //    scellement est celui de l'autorité gravée dans la genèse.
    let bloc1 = Bloc::from_bytes(&lire("bloc-1.bin")).expect("bloc 1 indécodable");
    assert_eq!(hex::encode(bloc1.id()), att["bloc1_id"], "id du bloc 1");
    let autorite = etat
        .producteur_attendu(1)
        .expect("chaîne à autorités attendue")
        .clone();
    assert!(
        bloc1.verifier_scellement(&autorite),
        "le scellement du bloc 1 n'est pas celui de l'autorité du tour"
    );

    // 4. Il s'applique, et l'état avance exactement comme publié.
    etat.appliquer_bloc(&bloc1).expect("bloc 1 refusé");
    assert_eq!(
        hex::encode(etat.tete()),
        att["bloc1_id"],
        "la tête n'a pas avancé jusqu'au bloc 1"
    );
    assert_eq!(
        hex::encode(etat.tree.root()),
        att["racine_apres_bloc1"],
        "racine après bloc 1 différente"
    );
}

    bloc1.signer_scellement(&autorite);

}
