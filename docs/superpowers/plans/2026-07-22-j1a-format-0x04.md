# J1-a — Format de bloc `0x04` : vue et certificat de quorum

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Donner au bloc la capacité de porter une **vue** et un **certificat de
quorum**, et de les vérifier — sans encore produire de votes.

**Architecture:** Le format et sa vérification d'abord, le protocole ensuite. Un
bloc de version `0x04` porte `vue: u32` (**dans** l'identifiant) et
`certificat: Option<Certificat>` (**hors** de l'identifiant, comme le scellement
— une signature sur l'identifiant ne peut pas y entrer). `appliquer_bloc` exige
`2f+1` signatures valides et distinctes avant tout coût STARK. Les tests
fabriquent les certificats directement : aucune machine à états n'est nécessaire
pour valider ce plan, ce qui le rend livrable seul.

**Tech Stack:** Rust 1.87, `crypto::sig` (signature hybride 3374 o), aucune
dépendance nouvelle.

**Références :** `docs/superpowers/specs/2026-07-22-j1-consensus-adr.md`
(ADR accepté), points 1, 2, 3 et 5.

## Global Constraints

- **Code et commentaires en français.**
- **Aucune dépendance ajoutée.**
- **`VERSION_BLOC` passe à `0x04`.** Un bloc `0x03` est **refusé par une variante
  d'erreur qui le nomme**, jamais réinterprété — discipline déjà appliquée à
  `CryptoError::AlgoPerime` et à `VERSION_ETAT`.
- **Tout décodage est borné AVANT allocation.** Le nombre de signatures est
  dérivé du masque, lui-même borné par `MAX_AUTORITES`.
- **La vue entre dans l'identifiant ; le certificat n'y entre pas.** Deux vues
  donnent deux blocs différents, jamais deux encodages du même.
- **`DOMAINE_VOTE` est distinct de `DOMAINE_SCELLEMENT`** : un scellement ne doit
  jamais pouvoir être rejoué comme un vote.
- **Ordre de vérification préservé** : contrôles O(1) → chaînage → certificat →
  **puis seulement** STARK. Le certificat coûte ~43 vérifications ML-DSA au pire ;
  il doit rester devant la preuve, jamais derrière.

### Task 2: le type `Certificat` et son encodage canonique

**Pourquoi un masque de bits plutôt qu'une liste d'index.** 8 octets pour 64
autorités, contre 1 à 2 octets par votant plus le risque de doublons. Le masque
rend les doublons **structurellement impossibles** : un bit est mis ou ne l'est
pas.

```rust
/// Domaine de signature d'un VOTE de quorum.
///
/// ⚠️ DISTINCT de `DOMAINE_SCELLEMENT`, et ce n'est pas cosmétique : sans cela,
/// le scellement d'un producteur — qui porte sur le même identifiant de bloc —
/// pourrait être compté comme l'un des votes du quorum. Un producteur atteindrait
/// alors `2f+1` avec `2f` votes réels.
pub const DOMAINE_VOTE: &str = "obscura/bloc/vote/v1";

/// Certificat de quorum : la preuve que `2f+1` autorités ont vu ce bloc à cette
/// `(hauteur, vue)`.
///
/// Les signatures sont rangées dans l'ORDRE CROISSANT des index de votants, et
/// leur nombre est DÉRIVÉ du masque — jamais annoncé séparément. Deux encodages
/// du même certificat sont donc impossibles.
///
/// ⚠️ Pas d'agrégation : aucune signature post-quantique ne l'offre (ADR J1).
/// Le certificat pèse `popcount(masque) × 3374` octets.
#[derive(Clone, Debug, PartialEq, Eq)]
pub struct Certificat {
    /// Bit `i` mis = l'autorité d'index `i` a voté. Les doublons sont
    /// structurellement impossibles.
    pub masque: u64,
    /// Une signature par bit mis, dans l'ordre croissant des index.
    pub signatures: Vec<HybridSignature>,
}

impl Certificat {
    /// Index des votants, croissants.
    pub fn votants(&self) -> impl Iterator<Item = usize> + '_ {
        (0..64).filter(move |i| self.masque & (1u64 << i) != 0)
    }

    pub fn to_bytes(&self) -> Vec<u8> {
        let mut b = self.masque.to_le_bytes().to_vec();
        for s in &self.signatures {
            let o = s.to_bytes();
            b.extend_from_slice(&(o.len() as u32).to_le_bytes());
            b.extend_from_slice(&o);
        }
        b
    }

    pub fn from_bytes(b: &[u8]) -> Result<Self, BlocDecodeError> {
        if b.len() < 8 {
            return Err(BlocDecodeError::Tronque);
        }
        let masque = u64::from_le_bytes(b[..8].try_into().expect("8 octets"));
        // Le NOMBRE est dérivé du masque : borné avant toute allocation.
        let attendu = masque.count_ones() as usize;
        if attendu > MAX_AUTORITES {
            return Err(BlocDecodeError::TropDAutorites { recues: attendu });
        }
        let mut signatures = Vec::with_capacity(attendu);
        let mut pos = 8;
        for _ in 0..attendu {
            if pos + 4 > b.len() {
                return Err(BlocDecodeError::Tronque);
            }
            let n = u32::from_le_bytes(b[pos..pos + 4].try_into().expect("4 octets")) as usize;
            pos += 4;
            if n > TAILLE_SCELLEMENT_MAX || pos + n > b.len() {
                return Err(BlocDecodeError::Tronque);
            }
            signatures.push(
                HybridSignature::from_bytes(&b[pos..pos + n])
                    .map_err(|_| BlocDecodeError::SignatureInvalide)?,
            );
            pos += n;
        }
        if pos != b.len() {
            return Err(BlocDecodeError::OctetsResiduels);
        }
        Ok(Certificat { masque, signatures })
    }
}
```

```rust
    ///
```

Élargir la signature :

```rust
    /// Producteur LÉGITIME de `(hauteur, vue)` : `autorites[(hauteur − 1 + vue) mod n]`.
    ///
    /// La vue décale le tour de rôle : c'est ce qui permet, quand l'autorité du
    /// tour ne répond pas, de désigner la suivante sans ambiguïté et sans que
    /// deux blocs concurrents soient tous deux « légitimes » — ils portent des
    /// vues différentes, donc des identifiants différents.
    pub fn producteur_attendu(&self, hauteur: u64, vue: u32) -> Option<&SigPublicKey> {
        if self.autorites.is_empty() || hauteur == 0 {
            return None;
        }
        let n = self.autorites.len() as u64;
        let i = ((hauteur - 1 + vue as u64) % n) as usize;
        Some(&self.autorites[i])
    }

    /// Quorum requis : `2f + 1` avec `n = 3f + 1`, soit `f = (n − 1) / 3`.
    pub fn quorum_requis(&self) -> usize {
        let n = self.autorites.len();
        if n == 0 {
            return 0;
        }
        2 * ((n - 1) / 3) + 1
    }
```

Dans `appliquer_bloc`, **après** le contrôle de chaînage et **avant** toute
vérification STARK :

```rust
        // CERTIFICAT DE QUORUM — après le chaînage (un bloc d'une autre chaîne ne
        // doit rien coûter) et AVANT tout STARK (le certificat coûte jusqu'à 43
        // vérifications ML-DSA ; la preuve en coûte davantage).
        if !self.autorites.is_empty() {
            let requis = self.quorum_requis();
            let Some(cert) = bloc.certificat.as_ref() else {
                return Err(BlocRefus::QuorumInsuffisant { obtenu: 0, requis });
            };
            let id = bloc.id();
            let mut obtenu = 0usize;
            for (rang, index) in cert.votants().enumerate() {
                let Some(pk) = self.autorites.get(index) else {
                    return Err(BlocRefus::VotantInconnu { index });
                };
                let sig = cert
                    .signatures
                    .get(rang)
                    .ok_or(BlocRefus::VoteInvalide { index })?;
                if !crypto::sig::verify(pk, DOMAINE_VOTE, &id, sig) {
                    return Err(BlocRefus::VoteInvalide { index });
                }
                obtenu += 1;
            }
            if obtenu < requis {
                return Err(BlocRefus::QuorumInsuffisant { obtenu, requis });
            }
        }
```

⚠️ **`cert.votants()` étant croissant et sans doublon par construction, `obtenu`
compte des autorités DISTINCTES.** C'est ce qui rend le comptage sûr, et c'est la
raison du masque.

## Critère de sortie de J1-a

- Un bloc porte une vue, et deux vues donnent deux identifiants.
- Un bloc `0x03` est refusé par une variante qui le nomme.
- Un certificat à `2f+1` signatures valides est accepté ; à `2f`, refusé.
- Un scellement rejoué comme vote est refusé.
- Le certificat est vérifié **après le chaînage et avant tout STARK**.
- La fixture v2 se rejoue ; la v1 a été supprimée après avoir échoué.
- Suite complète verte, clippy propre.

**Ce que J1-a ne fait PAS** : aucun vote ne circule sur le fil, aucun délai,
aucun changement de vue automatique. Les blocs restent produits par le tour de
rôle et les certificats sont fabriqués par les tests. **Le réseau ne gagne donc
pas encore la liveness** — c'est J1-b, et le dire évite de croire la porte D
close par ce plan.
